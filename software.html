Building the UI for the desktop of tomorrow

Desktop PC's haven't changed much over the years, we're still used to have these big boxes that occupy space, consume a lot of power and sometimes use hard to navigate interfaces. Much has been done to greatly improve this, better hardware that uses less power, smaller computer form factors and more useful interfaces, but while those advances are a good step-up from the past there's still much to be done. Speaking of advances, GUI-wise we've seen the rise of these interfaces that have set the way we interact with our computers. And each and every time these interfaces keep repeating the same formula over and over again, but we at NitruxÂ® are developing a new way of interacting with a desktop PC, one that doesn't alienate the user but rather embraces the natural interaction between man and machine.

The UI as it has been established will function for both types of current human-computer interaction, hardware based (keyboard and mouse) and touch based. Not in need of special gestures, movements or menus touch based interaction is no different than using the mouse and vice versa. The mouse brings precision while touch brings flexibility, these are the foundations for the UX. Keeping it simple, that's what we're all about.

Improving the interaction between humans and machines, with a new UI and UX that will focus on the reactions of the user and how computers respond to them. A scalable and flexible desktop environment, reasonable and sensible decisions targeting the interaction of the users and their applications.

Simple, quick and responsive, that's Nitrux

Our first foray into the world of Linux distributions, Nitrux® OS, as of version 1.0 is based on Ubuntu+1, this means it's based the development version of Ubuntu® and basically means we pack the latest software as it's available for it though we'll switch repositories before version 2.0 to our own while keeping binary compatibility with it with the intention of becoming a rolling-release distribution.

We have chosen the KDE SC as our default desktop (at version 4.13.x) as it provides a quick, responsive desktop environment all magnified by our custom resources from artwork to system tweaks which include performance tweaks for the hardware where it'll run.

With Nitrux® OS we have simplified the KDE desktop at the best of our abilities from the looks to its resource usage removing any bloatware from the base system and the desktop environment which has increased its performance appreciably.

Putting aesthetics aside, Nitrux® OS has been made with one thing in mind, performance. Only the most essential packages are installed by default removing completely those unnecessary bits that are absolutely irrelevant to the systems daily operations and the user's daily interactions.

Software choice has been left to a bare minimum, as we think users are likely to know better what they need installed. Muon Discover manages the software.

Extending the principles of minimalism throughout, no app escapes the applied design we have achieved with our custom resources. We've improved various aspects of the KDE desktop, including the Plasma workspace, resource usage, software selection and more.
